# 发布管理增强 — 设计规格

## 概述

本设计增强卡牌勘误平台的发布管理能力，将当前简陋的线性 Steps 页面改造为“勘误包 + 发布会话 + 可追溯产物”的发布中心。目标是让管理员在发布前能够清楚看到勘误包状态、精灵图内容、URL 替换结果、新增 TTS 对象落点和最终补丁包清单，并确保补丁包中的文件一定对应本次勘误包生成的卡图。

本设计不改变核心数据源边界：`SCED` 和 `SCED-downloads` 仍视为上游/目标仓库，不在系统中直接写入官方仓库；系统只导出可下载补丁包，由管理员复制到自己的 fork 后提交 PR。

## 背景与现状问题

当前发布管理页面存在以下问题：

1. **勘误包选择信息不足**：只有下拉框，包号、创建时间、更新时间、发布时间、卡牌数、发布状态、是否已回退等关键信息不可见。
2. **本地上传 URL 不可访问**：`LocalUploader` 返回 `http://localhost:8234/sheets/...`，但系统没有启动该端口对应的静态服务，导致 TTS 存档图片访问失败。
3. **TTS 存档原样上传识别 0 张卡**：当前导出的是单个 `Custom_Model_Bag` 对象，卡牌挂在动态键中；解析器却只扫描 `ObjectStates`，所以导出文件原样上传也无法识别卡牌。
4. **精灵图缺少发布前预览**：生成精灵图后无法先检查整张 front/back sheet 是否正确，只能继续上传。
5. **缺少回退上一步能力**：某一步发现错误只能重开，且后续产物没有明确废弃状态。
6. **URL 替换结果不可审阅**：管理员看不到每张卡原中文 TTS URL、新 URL、目标 JSON 路径和新增/替换动作。
7. **缺少新增卡牌落点规则**：如果某张勘误卡没有中文 TTS 记录，系统无法知道应该在哪个 Campaigns 或 Player Cards 包里新增对象。
8. **中间结果不可恢复**：刷新页面或换浏览器后，已生成的精灵图、URL 映射、补丁包路径等状态会丢失或难以继续。

## 目标

- 使用表格展示勘误包，提供足够的发布时间线和状态信息。
- 为每个勘误包创建独立发布会话，持久化每一步产物和状态。
- 支持精灵图预览、确认后继续、回退上一步、刷新后恢复。
- 修复 TTS JSON 解析，使 step3 导出的文件原样上传 step5 时能识别出同一批卡牌。
- 支持 URL 映射的审阅和校验，不再依赖不可访问的 `localhost:8234`。
- 补丁包必须从当前发布会话的已确认产物生成，不能信任前端传入任意 URL。
- 为新增中文 TTS 对象建立管理员可维护的发布目录预设表。
- 保证不同勘误包、不同发布尝试之间文件隔离，并提供清理策略。

## 非目标

- 不在本系统中直接修改 `SCED-downloads` 官方仓库。
- 不在本阶段实现完整云盘账号管理后台。
- 不允许多个待发布勘误包并行发布；沿用当前“同一时间只允许一个待发布或发布中勘误包”的产品约束。
- 不改变勘误副本的创建、审核、归档语义。

## 核心设计决策

| 决策 | 选择 |
|------|------|
| 发布状态承载 | 新增发布会话 `PublishSession`，而不是把所有中间状态塞进 `ErrataPackage` |
| 中间文件存储 | 文件落盘到 `data/cache/publish/<package_no>/<session_id>/`，数据库只存索引、路径、状态和校验信息 |
| URL 来源 | 支持 Steam/TTS JSON 识别、URL 映射 JSON 导入、表格手动填写三种输入 |
| Local URL | 只作为开发预览，统一使用后端 `/static/cache/...`；不能作为最终发布 URL 默认值 |
| 新增卡牌落点 | 使用管理员维护的 `PublishDirectoryPreset` 目录预设表，只影响中文 TTS 中不存在的新增卡牌 |
| 回退策略 | 回退会废弃后续产物，旧产物标记为 `superseded`，由清理任务异步删除 |
| 补丁包来源 | 只能从当前发布会话的 confirmed artifact 生成，不能由前端提交任意 card/url 数据 |

## 数据模型

### ErrataPackage 扩展展示字段

`ErrataPackage` 继续表示勘误包本身，不强行承载每一步发布产物。列表接口需要聚合展示：

- `card_count`：包内未归档勘误副本数量。
- `created_by_username`：创建人。
- `published_by_username`：发布人。
- `latest_session`：最近一次发布会话摘要。
- `artifact_summary`：是否已有精灵图、TTS 存档、URL 映射和补丁包。

这些字段可通过查询聚合返回，不一定全部入表。

### PublishSession

新增发布会话表，一次发布尝试对应一条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 主键 |
| `package_id` | integer | 关联 `errata_packages.id` |
| `status` | enum | `草稿`、`生成中`、`待确认精灵图`、`待准备URL`、`待导出补丁`、`已完成`、`已废弃`、`失败` |
| `current_step` | string | 当前步骤标识 |
| `artifact_root` | string | 会话文件根目录，相对 `project_root` |
| `created_by` | integer | 创建管理员 |
| `updated_by` | integer | 最近操作管理员 |
| `error_message` | text | 最近失败原因 |
| `cleanup_at` | datetime | 可清理时间 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

约束：同一个 `package_id` 同一时间只能有一个未完成且未废弃的发布会话。

### PublishArtifact

记录每一步产物。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 主键 |
| `session_id` | integer | 关联 `publish_sessions.id` |
| `kind` | enum | `card_image`、`sheet_front`、`sheet_back`、`tts_bag`、`url_mapping`、`patch_zip`、`manifest`、`report` |
| `status` | enum | `active`、`confirmed`、`superseded`、`deleted`、`failed` |
| `path` | string | 文件路径，相对 `project_root` |
| `public_url` | text | 前端预览 URL 或最终公开 URL |
| `checksum` | string | 文件 hash，用于一致性校验 |
| `metadata` | JSON | sheet 名、卡号列表、grid、目标路径、错误详情等 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

### PublishDirectoryPreset

管理员维护的新增 TTS 对象落点表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 主键 |
| `local_dir_prefix` | string | 本地 `.card` 相对目录前缀，使用最长前缀匹配 |
| `target_area` | enum | `campaigns` 或 `player_cards` |
| `target_bag_path` | string | SCED-downloads 中目标 Bag JSON 相对路径 |
| `target_bag_guid` | string | 目标 Bag GUID，例如 `8d1ac7` |
| `target_object_dir` | string | decomposed 子目录名，例如 `Core.8d1ac7` |
| `label` | string | 管理员可读名称，例如 `01_基础游戏 -> Core` |
| `is_active` | boolean | 是否启用 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

该表只用于“中文 TTS 中没有现有记录”的新增卡牌。已有中文 TTS 记录的卡牌始终沿用原 `relative_json_path` 更新。

## 发布流程

### 1. 勘误包选择

发布页顶部使用表格替代下拉框，展示：

- 包号。
- 状态：`待发布`、`发布中`、`已发布`、`已退回`。
- 卡牌数。
- 创建人、创建时间、更新时间、发布时间。
- 最近发布会话状态。
- 是否已有补丁包。
- 备注。

管理员可以对 `待发布` 包执行：

- 创建发布会话。
- 继续未完成会话。
- 解锁整包回到勘误状态。
- 查看发布前审阅工作台。

### 2. 生成并预览精灵图

后端根据勘误包中的 `ErrataDraft.modified_faces` 重新渲染卡图，生成：

- 每张卡的本地发布图。
- front 精灵图。
- 双面卡对应的 back 精灵图。
- sheet manifest：记录每张卡在 sheet 中的 grid 位置、arkhamdb_id、face、文件 hash。

前端展示：

- front/back 精灵图缩略图。
- 点击放大预览整张图。
- 下载单张 sheet。
- 卡牌清单与 grid 位置。
- 渲染失败或缺图异常。

管理员确认精灵图后，相关 `PublishArtifact` 状态改为 `confirmed`，才能进入 URL 阶段。

### 3. 准备公开 URL

此阶段不默认上传到不存在的本地 `8234` 服务。系统提供三种 URL 输入：

1. **TTS/Steam JSON 识别**：管理员下载 TTS 存档、在 TTS 中加载/处理后，再上传导出的 JSON，系统提取 Steam URL。
2. **URL 映射 JSON 导入**：管理员从其他云盘或脚本得到 `{sheet_name: url}` 或 `{arkhamdb_id: url}` 格式映射后导入。
3. **表格手动填写**：管理员直接在 URL 对照表中粘贴每张 sheet 的最终公开 URL。

无论 URL 来源如何，都统一写入当前 `PublishSession` 的 `url_mapping` artifact，并记录来源和校验结果。

### 4. 导出 TTS 存档

导出的 TTS JSON 必须能被系统自己解析。解析器需要支持：

- 标准 TTS save 格式：`ObjectStates`。
- 单个 `Custom_Model_Bag` 对象。
- 当前系统生成的动态键结构：`卡名.GUID`。
- decomposed 结构中的 `ContainedObjects_path`。
- 直接上传 `Card` JSON。

硬性验收：step3 导出的 TTS JSON 原样上传 step5，识别数量必须等于勘误包卡牌数，缺失时返回缺失卡号和解析路径说明。

### 5. URL 对照预览

进入补丁包导出前，前端展示每张卡的替换计划：

| 字段 | 说明 |
|------|------|
| 卡号 | arkhamdb_id |
| 名称 | 中文名 |
| 动作 | `替换` 或 `新增` |
| 原 JSON 路径 | 中文 TTS 当前文件路径；新增卡为空 |
| 目标 JSON 路径 | 补丁包内将写入的位置 |
| 原 FaceURL/BackURL | 当前中文包 URL |
| 新 FaceURL/BackURL | 本次发布 URL |
| sheet/grid | 新 URL 对应 sheet 和网格位置 |
| 校验结果 | 通过、缺 URL、缺中文 TTS、缺目录预设、checksum 不一致等 |

异常行置顶。存在阻断级异常时不能导出补丁包。

### 6. 导出补丁包

补丁包 ZIP 包含：

- `decomposed/language-pack/...` 下需要覆盖或新增的 JSON 文件。
- `manifest.json`：包号、会话 ID、卡牌清单、sheet 文件 hash、URL 映射、生成时间、生成用户。
- `url_mapping.json`：最终 URL 对照。
- `validation_report.json`：校验报告。

补丁包生成规则：

- 已有中文 TTS 记录：复制原 JSON，替换 `CardID`、`CustomDeck`、`FaceURL`、`BackURL`、`NumWidth`、`NumHeight`、`UniqueBack`，并保留必要的卡背 override。
- 新增中文 TTS 记录：根据 `PublishDirectoryPreset` 找目标 Bag，创建新 Card JSON，更新父 Bag 的 `ContainedObjects_order` 和 decomposed 子目录结构。
- 补丁包引用的所有新 URL 必须来自当前会话 confirmed artifact。
- 如果任一卡牌缺少 URL、缺少目录预设、或 checksum 与当前会话精灵图不匹配，导出失败。

### 7. 完成发布

管理员确认补丁包已用于自己的 fork 并提交 PR 后，可以点击“完成发布”：

- `ErrataPackage.status` 改为 `已发布`。
- 包内 `ErrataDraft.status` 改为 `已归档`，设置 `archived_at`。
- 保留补丁包 ZIP、manifest 和校验报告。
- 当前发布会话改为 `已完成`。

如果发布前发现问题，管理员可以整包回退：

- 从发布会话回退上一步：只影响当前会话，废弃后续 artifact。
- 解锁整包：`ErrataPackage.status` 改为 `已退回`，包内 draft 回到 `勘误`，可以继续由审核员修改。

## 新增卡牌目录预设

### 初始化规则

系统提供初始化脚本或管理端按钮：

1. 扫描 `卡牌数据库` 下的本地目录。
2. 扫描 `SCED-downloads/decomposed/language-pack/Simplified Chinese - Campaigns` 和 `Simplified Chinese - Player Cards` 的顶层 Bag。
3. 根据目录编号、名称和现有中文 TTS 卡牌位置生成初始 `PublishDirectoryPreset`。
4. 无法确定目标 Bag 的目录标记为未启用，并在发布页提示管理员配置。

### 匹配规则

- 新增卡牌通过本地 `.card` 的 `relative_path` 匹配 `local_dir_prefix`。
- 使用最长前缀匹配，避免 `剧本卡/09_绯红密钥` 被更宽泛的 `剧本卡` 规则覆盖。
- 匹配结果只决定新增对象写入位置，不影响已有中文 TTS 文件。

### 管理页面

发布管理中新增“发布目录索引”模块：

- 展示本地目录、目标区域、目标 Bag、目标对象目录、启用状态。
- 支持编辑目标 Bag 和对象目录。
- 展示该目录下本地卡牌数量、已有中文 TTS 数量、新增候选数量。
- 支持按目录名和包号搜索。

## TTS 解析修复

当前识别 0 张卡的根因是导出和解析结构不对称。设计中将 TTS 解析抽象为 `TTSObjectWalker`：

- 输入可以是保存文件、单 Bag、单 Card、decomposed 顶层 Bag。
- walker 会递归遍历：`ObjectStates`、`ContainedObjects`、动态键对象、`ContainedObjects_order` 对应对象。
- 对于 `ContainedObjects_path`，在有文件上下文时可以读取对应目录；上传文件没有目录上下文时只解析 JSON 内已有对象。
- 每个 `Card` 统一提取：`GMNotes.id`、`CardID`、`deck_key`、`FaceURL`、`BackURL`、`NumWidth`、`NumHeight`、`UniqueBack`。
- 返回结果包含解析来源路径，方便前端显示“从哪里识别到这张卡”。

## 文件隔离与清理

### 隔离目录

每个发布会话使用独立目录：

```text
data/cache/publish/
└── <package_no>/
    └── <session_id>/
        ├── cards/
        ├── sheets/
        ├── tts/
        ├── mappings/
        ├── patch/
        └── reports/
```

不同勘误包、不同发布尝试不会共用目录。生成补丁包时只读取当前 `session_id` 下的 confirmed artifact。

### 清理策略

- 默认保留每个勘误包最近一次成功会话。
- 默认保留最近 3 次失败或废弃会话。
- `superseded` artifact 超过 14 天后可自动删除。
- 已发布包的 `patch_zip`、`manifest`、`validation_report` 长期保留。
- 清理任务先更新数据库状态为 `deleted`，再删除文件；删除失败记录错误，不影响服务启动。
- 管理端提供“清理废弃产物”按钮，但只清理已标记可清理的 artifact。

## API 设计

### 勘误包

- `GET /api/admin/packages`：返回增强后的包列表。
- `GET /api/admin/packages/{id}`：返回包详情、草稿列表、最近会话摘要。

### 发布会话

- `POST /api/admin/publish/sessions`：为包创建发布会话。
- `GET /api/admin/publish/sessions/{id}`：获取会话、步骤、artifact、校验摘要。
- `POST /api/admin/publish/sessions/{id}/generate-sheets`：生成并记录精灵图。
- `POST /api/admin/publish/sessions/{id}/confirm-sheets`：确认精灵图。
- `POST /api/admin/publish/sessions/{id}/export-tts`：导出 TTS 存档。
- `POST /api/admin/publish/sessions/{id}/import-urls`：上传 TTS JSON 或 URL 映射 JSON。
- `PATCH /api/admin/publish/sessions/{id}/urls`：手动修改 URL 映射。
- `GET /api/admin/publish/sessions/{id}/replacement-preview`：获取 URL 对照和补丁计划。
- `POST /api/admin/publish/sessions/{id}/export-patch`：导出补丁包。
- `POST /api/admin/publish/sessions/{id}/rollback-step`：回退上一步并废弃后续 artifact。
- `POST /api/admin/publish/sessions/{id}/complete`：完成发布并归档勘误包。

现有 `step1` 至 `step6` API 可保留为兼容层，内部委托到会话 API；前端迁移完成后再删除旧接口。

### 发布目录预设

- `GET /api/admin/publish/directory-presets`：列表。
- `POST /api/admin/publish/directory-presets/initialize`：从当前仓库结构初始化。
- `PATCH /api/admin/publish/directory-presets/{id}`：修改目标 Bag 或启用状态。
- `GET /api/admin/publish/directory-presets/coverage`：查看目录覆盖率和新增候选。

## 前端设计

发布页分为三块：

1. **勘误包表格**：显示包状态、时间线、卡牌数、最近会话、操作按钮。
2. **发布会话向导**：展示当前步骤内容，支持上一步、下一步、重新生成、确认。
3. **发布前审阅工作台**：复用现有 `CardWorkbench mode="package-review"`，用于管理员发布前查看勘误卡牌内容。

### 关键 UI

- 精灵图预览：front/back 卡片式缩略图，点击放大，显示文件大小、hash、卡牌数、grid。
- URL 对照表：异常置顶，可筛选 `缺 URL`、`新增卡`、`替换卡`、`checksum 异常`。
- 补丁包结果：显示 ZIP 下载、manifest 下载、校验报告下载和下一步说明。
- 目录预设表：单独作为发布管理中的配置页签，避免和普通勘误用户页面混在一起。

## 错误处理

- 生成精灵图失败：记录到 `PublishSession.error_message`，失败 artifact 标记 `failed`，前端展示失败卡号。
- URL 识别失败：返回识别数量、期望数量、缺失卡号、解析器扫描过的结构类型。
- 新增卡牌缺目录预设：阻断补丁包导出，提示进入“发布目录索引”配置。
- 本地预览 URL 缺文件：返回明确错误，不再生成不可访问的 `localhost:8234` URL。
- 回退失败：不删除文件，仅更新会话错误状态，避免数据丢失。

## 测试计划

### 后端测试

- `step3` 导出的 TTS JSON 原样上传解析，识别数量等于包内卡牌数。
- 解析器支持 `ObjectStates`、单 Bag、单 Card、动态键结构。
- 发布会话目录隔离：两个包生成的 sheet 不互相覆盖。
- 回退上一步会将后续 artifact 标记为 `superseded`。
- 补丁包只使用当前会话 confirmed artifact。
- 新增卡牌命中目录预设后生成 Card JSON 并更新父 Bag order。
- 新增卡牌缺目录预设时导出失败。
- 清理任务不会删除最新成功会话的补丁包和 manifest。

### 前端测试

- 勘误包表格展示状态、时间、卡牌数和最近会话。
- 精灵图预览能显示 front/back 缩略图并放大。
- 上一步按钮会提示将废弃后续产物。
- URL 对照表异常行置顶。
- 原样上传识别 0 张时展示缺失原因，而不是只显示“提取 0 张”。

## 迁移策略

1. 添加数据库迁移：`publish_sessions`、`publish_artifacts`、`publish_directory_presets`。
2. 新增发布会话服务层，保留旧 publish API 作为兼容层。
3. 修复 TTS 解析器，使现有导出文件可被原样识别。
4. 改造前端发布页为表格 + 会话向导。
5. 增加目录预设初始化和管理页面。
6. 完成补丁包 manifest 和校验报告。
7. 通过测试后隐藏或删除旧 step API 的直接使用入口。

## 验收标准

- 管理员能在发布页看到每个勘误包的状态、时间、卡牌数、最近会话和补丁包状态。
- 生成精灵图后必须能预览 front/back sheet，再决定是否进入下一步。
- `step3` 导出的 TTS 存档原样上传后，不再出现识别 0 张；识别数量必须等于勘误包卡牌数。
- URL 对照页能展示每张卡原 URL、新 URL、目标 JSON 路径和新增/替换动作。
- 没有中文 TTS 记录的新增卡牌必须通过目录预设决定落点；缺预设时阻断导出。
- 补丁包 ZIP 中的 JSON 只引用当前发布会话确认过的精灵图 URL。
- 刷新页面后可以继续当前发布会话。
- 回退上一步不会误删文件，后续产物会被标记废弃并可清理。
- 不同发布包和不同发布会话的文件互不覆盖。
